from sklearn.datasets import load_breast_cancer
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split

cancer = load_breast_cancer()

# Split the data
X_train, X_test, y_train, y_test = train_test_split(cancer.data, cancer.target, stratify=cancer.target, random_state=42)

tree = DecisionTreeClassifier(random_state=0)
tree.fit(X_train, y_train)

Il s'agit d'un overfitting. Cela se produit parce que l'arbre de dcision n'tait pas limit et est descendu jusqu' ce qu'il s'adapte parfaitement aux donnes d'apprentissage.

Pour viter l'Overfitting, nous appliquons des restrictions pour limiter la profondeur de l'arbre de dcision, ce qui son tour va limiter sa complexit.

Technique d'lagage :

Il existe deux types d'lagage, pr et post.

  • Avec l'lagage prcoce, nous arrtons la cration de l'arbre un stade plus prcoce. Ainsi, nous pouvons limiter :
  1. la profondeur de l'arbre,
  2. le nombre maximal de feuilles,
  3. spcifier un minimum de points pour un nud afin de pouvoir effectuer de nouvelles divisions.

En limitant la profondeur de l'arbre, nous limitons le nombre de questions pouvant tre poses.

  • Avec l'lagage postrieur, une fois l'arbre cr, nous pouvons supprimer les branches ou nuds non pertinents qui contiennent peu d'informations.

Avantages des arbres de décision¶

  • Facile visualiser et comprendre.
  • Contrairement d'autres algorithmes de classification par apprentissage automatique, il n'est pas ncessaire de prtraiter, de normaliser, de mettre l'chelle et/ou de standardiser les caractristiques.

Paramètres à utiliser¶

  • max_depth,
  • min_samples_leaf, max_samples_leaf,
  • max_leaf_nodes,
  • etc.

Principaux inconvénients¶

  • Tendance l'overfitting, mme si l'on cre un arbre d'lagage.
  • Mauvaise gnralisation.
